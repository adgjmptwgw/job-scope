# 画面詳細設計：求人詳細画面 (Detail Screen)

## 1. 概要
- **目的**: 選択された求人の詳細情報、AI評価の根拠、企業の実態評価を詳しく表示する。
- **パス**: `/detail/:id`

## 2. 構成要素 (UI Components)
- **メイン求人情報 (Job Info Card)**:
  - タイトル、企業名、勤務地、給与情報。
  - お気に入り登録・解除ボタン (Heart Icon): 画面上部のセクションに配置。
- **仕事内容・応募要件**:
  - セクション分けされた詳細テキスト。チェックアイコン等の装飾を使用。
- **企業実態評価 (Structured Evaluation Card)**:
  - **最重要セクション**: AI検索条件に基づき、AIが項目を分割して表示。
  - カテゴリ表示 (例: 残業時間、社員同士の仲、有給の取りやすさ)。
  - 項目ごとの根拠リンク (ExternalLink Icon付きのアンカータグ)。
- **AI評価分析 (AI Analysis Card)**:
  - AI評価スコア (5.0点満点) と星評価。
  - 評価理由 (aiReason) のテキスト表示。

## 3. 機能・ロジック
- **動的データ取得**:
  - URLの `id` パラメータに基づき、`mockJobs` から対象データを検索し状態にセット。
  - データ取得中はスケルトン/ローディングアニメーションを表示。
- **構造化された実態評価のレンダリング**:
  - `evaluationItems` 配列をマップし、各項目のカテゴリ名と複数のリンクをリスト形式でレンダリング。
- **お気に入り同期**:
  - 画面上での操作が検索画面やお気に入り一覧画面と即座に同期される。

## 4. 遷移

## 5. データ要件
- `mockData.ts` の `evaluationItems` を含む詳細なジョブオブジェクト。
- `evaluationItems` の形式: `{ category: string, links: string[] }[]`
