# 画面詳細設計：ログイン画面 (Login Screen)

## 1. 概要
- **目的**: ユーザー認証を行い、アプリケーションへのアクセスを許可する。
- **パス**: `/login` (デフォルト画面)

## 2. 構成要素 (UI Components)
- **カードレイアウト**: `shadcn/ui` の Card を使用した中央配置のデザイン。
- **入力フィールド**:
  - メールアドレス (Input): メールアドレスの入力。
  - パスワード (Input[type=password]): パスワードの入力。
- **ボタン**:
  - ログインボタン (Button): 認証実行。グラデーションスタイルの `variant="gradient"` を採用。
- **装飾要素**:
  - `framer-motion` を利用した浮遊感のある背景グラデーションとアニメーション。
  - 上部のシマーアニメーション付きプログレスライン。

## 3. 機能・ロジック
- **認証処理**:
  - **Supabase Auth Integration**: バックエンドAPI (`/api/auth/login`) を介して Supabase Auth で認証を行う。
  - 成功時：`求人検索画面 (/search)` へ遷移。
  - 失敗時：エラーメッセージを表示。
- **インタラクション**:
  - Enterキー押下でログイン処理を実行可能。
  - 各入力フィールドとボタンにフェードイン・スライドインのアニメーション（`framer-motion`）。

## 4. 遷移
- **ログイン成功**: → [求人検索画面](./02-search-screen.md)

## 5. データ要件
- Supabase Auth のセッション管理に準拠。
