# 画面詳細設計：アカウント設定画面 (Account Settings Screen)

## 1. 概要
- **目的**: ユーザーのプロフィール情報、メールアドレス、およびログインパスワードを管理し、セキュリティを維持する。
- **パス**: `/settings`

## 2. 構成要素 (UI Components)
- **ナビゲーション (Navigation)**:
  - サイドバーには表示しない。
  - ヘッダーまたは特定の導線（将来実装）からのアクセスとする。現状は直接アクセス (`/settings`) または開発者用導線のみ。
- **プロフィールセクション (Profile Section)**:
  - アバター画像 (Avatar with upload button): プロフィール写真の表示と変更。
  - 表示名 (Input): アプリ内で使用される名前。
- **アカウント情報セクション (Account Info Section)**:
  - 現在のメールアドレス (Disabled Input): 確認用。
  - 新しいメールアドレス (Input): 変更用。
- **セキュリティセクション (Security Section)**:
  - パスワード変更ボタン (Button): 変更用フォーム（ポップアップまたは展開式）を表示。

- **保存ボタン (Footer Button)**:
  - 変更内容を確定する。成功時にトースト通知を表示。
- **危険区域 (Danger Zone)**:
  - アカウント削除ボタン (Button, variant="destructive"): 退会処理。確認ダイアログを表示。

## 3. 機能・ロジック
- **入力バリデーション**:
  - メールの形式チェック。
  - パスワードの強度チェック。
- **変更確認**:
  - メアドやパスワードの変更時には、現在のパスワードの再入力を求める設計（セキュリティ強化）。
- **ステート管理**:
  - 変更がある場合のみ保存ボタンを活性化する。

## 4. 遷移
- **サイドバー**: [求人検索画面](./02-search-screen.md) 等へのナビゲーション。フッターのユーザー情報エリアをクリックしてアクセス可能。
- **ログアウト**: サイドバー下部のログアウトボタンから実行。[ログイン画面](./01-login-screen.md) へ戻る。

## 5. データ要件
- `User` オブジェクトの各フィールドを更新。
- デモ版では、ローカルストレージまたはモックステートを更新。
