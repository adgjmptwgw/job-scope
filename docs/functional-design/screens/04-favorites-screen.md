# 画面詳細設計：お気に入り一覧画面 (Favorites Screen)

## 1. 概要
- **目的**: ユーザーが保存した求人を一元管理し、クイックにアクセスできるようにする。
- **パス**: `/favorites`

## 2. 構成要素 (UI Components)
- **ヘッダーエリア**: 「保存済み求人」のバッジとタイトルを表示。
- **求人一覧グリッド**: 
  - 検索画面と同様の求人カードを表示。
  - カード上のハートアイコン（削除ボタン）を配置。
- **空状態表示 (Empty State Card)**: 保存済み求人がない場合のプレースホルダー。求人検索画面への誘導ボタンを含む。

## 3. 機能・ロジック
- **リアルタイム更新**:
  - ハートアイコンをクリックして「お気に入り解除」された場合、`AnimatePresence` を用いてカードがリストからスムーズに消去される演出。
  - 画面遷移なしで即座に表示が更新される。
- **データ取得**:
  - 起動時に `getFavoriteJobs()` を呼び出し、現在保存されているIDに対応する求人データを抽出して表示。

## 4. 遷移
- **カードクリック/詳細を見る**: → [求人詳細画面](./03-detail-screen.md)
- **空状態からの誘導**: → [求人検索画面](./02-search-screen.md)

## 5. データ要件
- `mockData.ts` の `favoriteJobIds` および `getFavoriteJobs()` 関数を利用。
