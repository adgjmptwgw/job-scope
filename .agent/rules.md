# Antigravity Rules

このプロジェクトにおけるAntigravity（AIエージェント）の行動指針です。

## 1. ドキュメントの運用
- **実装前**: 必ず関連するドキュメント（要件定義書、API仕様書、設計書など）を読み込み、コンテキストを把握すること。
- **実装後**: 実装内容に合わせて、関連ドキュメントを必ず更新すること。

## 2. コードコメント
- **関数・クラス**: すべての関数やクラスには、以下の要素を含む説明文を記載すること。
    - **What**: 何をするものか（機能の概要）
    - **Why**: なぜその実装が必要か、なぜそのアプローチを選んだか（意図や背景）

## 3. 言語設定
- **コメント**: コード内のコメントはすべて**日本語**で記述すること。
- **回答**: ユーザーへの回答や対話はすべて**日本語**で行うこと。

## 4. 行動指針・制約
- **仕様の厳格な管理**: 仕様を厳格に管理することがプロジェクトにおいて最も重要である。
- **指示外の行動の禁止**: ユーザーから指示された内容以上のことを勝手に実施してはならない（過剰なリファクタリング、未指示の機能追加など）。
- **例外**: コンパイルエラーや重大なバグなど、修正しなければシステムが動作しない「絶対に修正すべき点」に限り、指示なしで修正を行ってもよい。
